## Summary
### MoonRay Technical Steering Committee Welcome:
The meeting focused on welcoming Michel Castejon as the first external member of the MoonRay Technical Steering Committee. Michel, a developer at Apple's Metal graphics team, was recognized for his significant contributions to MoonRay, including the macOS port, build system, and Metal XPU support.

### MoonRay TSC Expansion Discussion:
The team discussed expanding the Technical Steering Committee (TSC) for MoonRay, with Jon proposing to adopt a structure similar to MaterialX's model that distinguishes between voting members and stakeholders. Jon explained that stakeholders would serve as appointed representatives from companies or teams with significant interest in the project, allowing the TSC to gather feedback from key downstream users without requiring them to be full contributors.

### MoonRay TSC Meeting Notes:
The team also discussed meeting documentation processes, with Jon proposing to create markdown files for TSC meetings in the Open MoonRay repository and temporarily share notes in the MoonRay channel until GitHub migration to ASWF is complete. Jean-Francois explained that ASWF projects have access to Confluence spaces and suggested using hackmd.io for Markdown meeting notes. Jon will look into putting the notes in the MoonRay project's Confluence page and/or publishing them to the PCC, calendar, and individual dashboards.

### ASWF EasyCLA Implementation Planning:
The team discussed EasyCLA implementation and GitHub migration plans. Matthew clarified that existing MoonRay CLA signatories will need to re-sign through the new ASWF EasyCLA system once repositories are migrated to ASWF management.

### Super-project vs. Mono-project:
Jon outlined the current GitHub structure with 20-21 repositories and proposed consolidating them into a single repository (or a smaller set of repositories) in the future, though this would be disruptive to their current build process, so they plan to proceed with individual repositories for now.

### Open MoonRay GitHub Organization Planning:
The team discussed creating a GitHub organization for the Open MoonRay project under the Academy Software Foundation (ASWF). Matthew explained that while most small projects use ASWF's GitHub organization, larger projects can create their own organizations while still benefiting from ASWF's GitHub Enterprise features. Jean-Francois noted both benefits and drawbacks of separate organizations, including easier secret management but potentially reduced discoverability.

### MoonRay CI/CD Implementation Discussion:
The team discussed implementing CI/CD processes for MoonRay, focusing on how to test the renderer across different VFX platform versions. Jon outlined their current internal testing approach, which includes unit tests and a render test suite (RATS) that runs against canonical images on specific machine classes. The group explored options for public CI, including using GitHub Actions runners, self-hosted runners, or AWS services, with Jean-Francois suggesting the ASWF Docker project as a potential solution. They decided to initially target a modern VFX platform version (likely 2024 or 2025) rather than maintaining older versions like 2022, and agreed to revisit the topic of larger refactor projects like MaterialX for public visibility once the GitHub repositories are migrated.

### DWA MoonRay Development Process:
Jon outlined his vision for transitioning from DreamWorks' current internal development process to a public-first approach, where community contributions would be merged directly into the main branch, with DreamWorks maintaining read-only mirrors and separate integration branches for production work. Matthew emphasized the importance of establishing reliable CI and regression tests in the public domain to ensure smooth community contribution acceptance.
